# Format: docs/impl-notes/_.html One file per spec per iteration. Never overwrite an existing file. Multiple iterations of the same module produce multiple timestamped files — that is correct behaviour. ## Filename derivation ``` YYYY-MM-DDTHH-MM-SS_.html ``` **Timestamp** — local timezone, derived at the moment the `` is received: ```bash date '+%Y-%m-%dT%H-%M-%S' ``` **Slug** — in priority order: 1. `title` attribute if present: `` → `auth-module` 2. First non-empty line of the spec content, lowercased, spaces/punctuation → hyphens, max 48 chars Examples: ``` 2026-05-18T20-56-00_auth-module.html 2026-05-18T21-30-00_payment-flow-redesign.html 2026-08-12T09-15-00_auth-module.html ← second iteration, same module ``` ## Index file After creating or updating any impl-notes file, regenerate `docs/impl-notes/index.html`. Read all `*.html` files in the directory (excluding index.html itself), extract ``, status meta, and started meta from each, then write the index newest-first. Index template: ```html <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Implementation Notes — Index

Implementation Notes — Index

{{COUNT}} spec(s) · last updated {{UPDATED}}
{{ENTRIES}} ``` Each `{{ENTRIES}}` row: ```html
{{STARTED}} {{SPEC_TITLE}} {{STATUS}}
``` ## impl-notes file template Produce this structure for each new spec. Fill all `{{PLACEHOLDERS}}`. Inline all styles — file must open standalone in a browser. ```html impl-notes — {{SPEC_TITLE}}

{{SPEC_TITLE}}

{{SESSION_ID}} {{STARTED}} docs/impl-notes/{{FILENAME}} {{STATUS}}

Spec

{{SPEC_CONTENT}}

Design Decisions

No decisions recorded yet.

Deviations

No deviations recorded yet.

Tradeoffs

No tradeoffs recorded yet.

Open Questions

No open questions.

``` ## Field reference | Field | Value | |-------|-------| | `{{SPEC_TITLE}}` | `title` attribute if present; otherwise first non-empty line of spec, max 48 chars | | `{{SLUG}}` | `{{SPEC_TITLE}}` lowercased, spaces/punctuation → hyphens | | `{{FILENAME}}` | `YYYY-MM-DDTHH-MM-SS_{{SLUG}}.html` using local timezone | | `{{STARTED}}` | Local timezone datetime: `date '+%Y-%m-%d %H:%M:%S %Z'` | | `{{SESSION_ID}}` | `$CLAUDE_SESSION_ID` or "unknown" | | `{{STATUS}}` | `in-progress` → `complete` → `blocked` | | `{{SPEC_CONTENT}}` | Full spec verbatim, unmodified | | `{{TIMESTAMP}}` | Local timezone datetime of each entry | ## Update rules - **Never overwrite** an existing impl-notes file — each `` invocation creates a new timestamped file - **Update in-place** within the current session's file — sections are edited, not appended - **Remove `.empty` placeholders** as real entries are added to each section - **Resolve questions** by changing `badge-open` → `badge-resolved`, adding answer inline - **Set status `complete`** when implementation is done and all questions resolved or explicitly deferred - **Regenerate index.html** after every create or status update
