# Claude Safe-Yolo Mode — Allowed Tools
# ─────────────────────────────────────────────────────────────────────────────
# One entry per line. Blank lines and lines starting with # are ignored.
# This file is the single source of truth for what enable-safe-yolo manages.
# Synced to ~/.claude/claude-safe-yolo-permissions.txt by install.sh (from scripts/config/)
#
# IDEMPOTENT:
#   enable-safe-yolo  — only ADDS entries from this file not already present.
#   disable-safe-yolo — only REMOVES entries that are listed in this file.
#   Manually added permissions in .claude/settings.json are NEVER touched.
#
# FORMAT:
#   Tool(*)           — auto-approve all uses of a Claude Code built-in tool
#   Bash(command *)   — auto-approve Bash commands starting with "command " (space)
#   Bash(command:*)   — auto-approve Bash commands starting with "command" (prefix wildcard)
#   Both forms are included for each command to cover all invocation styles.
#   !Bash(command *)  — hard-BLOCK this Bash prefix (leading ! = deny rule);
#                       Claude cannot run these even if the user approves them
#   @defaultMode=VAL  — set permissions.defaultMode (e.g. acceptEdits)
#
# SCOPE:
#   enable-safe-yolo writes to <repo>/.claude/settings.json.
#   Claude Code reads that file only when launched inside that repo directory.
#   These permissions have NO effect outside that repo session.
#
# USAGE:
#   enable-safe-yolo                    # targets current working directory
#   enable-safe-yolo --dir=PATH         # targets a specific repo
#   enable-safe-yolo --dry-run          # preview only, no changes
#   disable-safe-yolo                   # removes only these entries from repo
#   disable-safe-yolo --dir=PATH        # targets a specific repo
#   /enable-safe-yolo                   # from inside a Claude Code session
#   /disable-safe-yolo                  # from inside a Claude Code session
# ─────────────────────────────────────────────────────────────────────────────
